## 1. 系统概述

本 ERP 系统覆盖企业核心业务流程：

| 模块 | 主要能力 |
|------|----------|
| 采购 | 供应商、采购申请、采购订单、收货验收 |
| 销售 | 客户、报价、销售订单、发货、售后工单、公海线索、联系上报与审核、**外勤 Android App** |
| 库存 | 台账、出入库、调拨、盘点、预警、批次/序列号追溯 |
| 生产 | BOM、生产计划、工单、质检 |
| 财务 | 总账、应收/应付、固定资产、预算、财务报表 |
| 人力 | 员工、部门、岗位、考勤、薪资、绩效 |
| 项目 | 项目与任务进度 |
| 报表 | 运营概览、BI 大屏、智能补货/财务分析 |
| 系统 | 用户、角色、权限、租户、审计、文件、集成 |

技术栈：React 前端 + NestJS 后端 + PostgreSQL，支持 Redis 缓存、MinIO 文件存储、WebSocket 通知。

## 2. 访问与登录

### 2.1 访问地址

| 环境 | 前端地址 | 后端 API |
|------|----------|----------|
| 开发模式 | http://localhost:5173 | http://localhost:3001/api（或见 `backend/.env` 中 `PORT`） |
| Docker Compose | http://localhost:8080 | http://localhost:3000/api |

### 2.2 登录步骤

1. 打开浏览器访问上述前端地址，未登录时会自动跳转到登录页。
2. 填写 **租户**（默认选 `default` / 默认租户）。
3. 填写 **用户名** 和 **密码**。
4. 点击 **登录**。

登录页右上角可切换 **中文 / English**。

### 2.3 默认与演示账号

执行数据库 seed 后会自动创建管理员与多角色演示用户。密码在 `backend/.env` 中通过以下变量配置（**切勿使用默认值部署到公网**）：

| 变量 | 说明 |
|------|------|
| `SEED_ADMIN_PASSWORD` | 管理员 `admin` 的初始密码 |
| `SEED_DEMO_PASSWORD` | 各角色演示用户的统一初始密码 |

| 账号 | 角色 | 用途 |
|------|------|------|
| `admin` | 系统管理员 | 全功能管理 |
| `employee` | 普通员工 | 权限测试（采购申请、库存只读、报表） |
| `finance_director` | 财务总监 | 财务全盘 |
| `finance_clerk` | 财务专员 | 日常记账 |
| `procurement_director` | 采购总监 | 采购全流程 |
| `procurement_clerk` | 采购专员 | 采购执行 |
| `sales_director` | 销售总监 | 销售全盘、上报审核 |
| `sales_clerk` | 销售专员 | 报价/订单/发货、公海领取、联系上报 |
| `production_manager` | 生产经理 | 生产+库存协调 |
| `production_clerk` | 生产专员 | 工单执行 |
| `warehouse_admin` | 仓库管理员 | 库存与追溯 |
| `hr_clerk` | 人事专员 | 人事模块 |

> seed 完成后控制台会列出已创建的用户名；密码以 `.env` 配置为准。

## 3. 界面布局与通用操作

登录成功后进入主界面，由四部分组成：

![数据看板](images/erp-dashboard.png)

### 3.1 左侧导航栏（Sider）

- 按业务域分组：财务、采购、销售、生产、库存、人力等。
- **仅显示当前用户有权限的菜单**；无权限的模块不会出现。
- 窄屏（手机/平板）下自动变为 **Drawer 抽屉菜单**，点击左上角图标展开。

### 3.2 顶栏（Header）

| 元素 | 说明 |
|------|------|
| 面包屑 | 显示当前页面路径，如 `首页 / 采购管理 / 采购申请` |
| 区域设置 | `Shanghai · CNY` 按钮，切换时区与币种（会同步到用户 Profile） |
| 语言 | 中文 / English |
| 通知铃铛 | 站内通知列表，支持 WebSocket 实时推送 |
| 用户名 | 当前登录用户姓名，点击可退出登录 |

### 3.3 主内容区

- 页面标题由 **PageTitle** 组件统一展示。
- 列表页普遍采用 **ResponsiveTable**：宽屏表格、窄屏自动卡片化，便于手机浏览。
- 常见按钮：**新建**、**编辑**、**删除**、**审批**、**刷新** 等，操作结果以右上角 Message 提示。

### 3.4 通用表格操作

1. 点击 **新建** 打开表单弹窗（Modal）。
2. 填写必填项（带红色星号），点击 **保存/创建**。
3. 行内 **编辑** 修改记录；**删除** 需二次确认。
4. 带审批流的单据需先 **提交/审批** 才能进入下一环节。

### 3.5 退出登录

点击 Header 右侧用户名 → **退出登录**。Token 清除后需重新登录。

## 4. 角色、账号与权限

### 4.1 权限模型

- 系统预置 **41 项权限**，按模块分组（system、finance、procurement、sales 等）。
- 每个 **角色** 绑定若干权限；用户通过 **角色** 获得权限。
- API 与菜单均受权限控制：
  - **菜单**：侧栏隐藏无权限项。
  - **路由**：直接访问无权限 URL 显示 403 页面。
  - **接口**：无权限返回 HTTP 403。

### 4.2 预置角色（12 个）

| 角色 | 权限数 | 典型使用人 |
|------|--------|------------|
| 系统管理员 | 41（全部） | IT / 实施顾问 |
| 财务总监 | 7 | 财务负责人 |
| 财务专员 | 5 | 会计 |
| 采购总监 | 7 | 采购经理 |
| 采购专员 | 4 | 采购员 |
| 销售总监 | 9+ | 销售经理（含线索审核） |
| 销售专员 | 8+ | 销售员（含公海与上报） |
| 生产经理 | 8 | 车间主任 |
| 生产专员 | 4 | 计划员/操作工 |
| 仓库管理员 | 5 | 仓管 |
| 人事专员 | 5 | HR |
| 普通员工 | 4 | 一线员工 |

### 4.3 配置角色权限

路径：**系统管理 → 角色管理**

![角色管理](images/erp-system-role.png)

**操作步骤：**

1. 在列表中找到目标角色，点击 **权限** 按钮。
2. 在弹窗中按模块勾选权限（finance、inventory、sales 等分组展示）。
3. 点击 **确定** 保存；系统会替换该角色的全部权限绑定。
4. 已登录用户需 **重新登录** 后权限才会刷新（JWT 内不含实时权限变更）。

**新建角色：**

1. 点击 **+ 添加角色**。
2. 填写角色名称、描述 → 创建。
3. 再点击 **权限** 分配具体权限。

### 4.4 用户管理

路径：**系统管理 → 用户管理**

1. **新建用户**：用户名、姓名、邮箱、角色、初始密码。
2. **编辑**：可修改角色、状态（启用/禁用）。
3. 用户被禁用后无法登录。

### 4.5 权限测试建议

用 `employee` 账号（演示密码见 `.env` 中 `SEED_DEMO_PASSWORD`）登录验证：

- 侧栏应仅有：采购申请、部分库存、报表中心等。
- 访问 `/system/role` 应显示 403。
- 可创建采购申请；**不能** 执行出入库调整（403）。

运行自动化审计（可选）：

```powershell
node tests/permissions-audit.mjs
```

## 6. 采购管理

### 6.1 业务流程总览

```
采购申请 → 审批 → 转采购订单 → 审批采购单 → 收货入库 → 应付生成
```

### 6.2 供应商管理

路径：**采购管理 → 供应商管理**

- 维护供应商编码、名称、联系人、电话。
- 采购订单创建时需选择供应商。

### 6.3 采购申请

路径：**采购管理 → 采购申请**

![采购申请](images/erp-procurement.png)

**新建申请：**

1. 点击 **+ 新建申请**。
2. 填写标题、申请原因。
3. 添加明细行：选择产品、填写数量、预估单价（可选）。
4. 保存后状态为 **草稿**，可编辑。
5. 点击 **提交审批** → 审批通过后可用 **转采购单** 生成采购订单。

**智能补货转申请：**

在 **报表中心 → 智能分析 → 智能补货** 中，可 **一键全部转采购申请** 或按行 **转采购申请**（需 `procurement:request` 权限）。

### 6.4 采购订单

路径：**采购管理 → 采购订单**

1. 由申请转换或手工新建。
2. 选择供应商、明细、单价。
3. **审批** 通过后进入可收货状态。
4. 审批时会 **扣减采购预算**（如已配置预算）。

### 6.5 收货验收

路径：**采购管理 → 收货验收**（与 **采购订单** 页共用同一功能，已审批订单均可在此收货）

![采购订单 — 收货](images/erp-procurement-receive.png)

1. 在 **采购订单** 或 **收货验收** 列表中找到状态为 **已审批（approved）** 的订单。
2. 点击 **收货**，在弹窗中选择 **收货仓库**（默认主仓库）。
3. 确认后 **库存增加**，并自动生成 **应付单**（详见 [10.3 应付管理](#103-应付管理)）。

> 订单须先 **审批** 才能收货；收货后状态变为 **已完成（completed）**，不可重复收货。

### 6.6 智能补货全流程

路径：**报表中心 → 智能分析 → 智能补货**

![智能补货](images/erp-intelligence-replenishment.png)

当产品 **当前库存 < 安全库存** 时，系统根据近 30 天出库速度与 lead time 计算建议采购量，可按以下链路闭环：

```
智能补货预警 → 转采购申请 → 审批 → 转采购订单 → 审批 → 收货 → 应付 → 付款 → 财务报表
```

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 产品管理：设置 **安全库存**（或库存低于阈值） | 智能补货 Tab 出现预警行 |
| 2 | 点击 **转采购申请** 或 **一键全部转采购申请** | 采购申请列表新增草稿/待审单 |
| 3 | 采购申请 → **提交审批** → **审批通过** → **转采购单** | 生成采购订单（草稿） |
| 4 | 采购订单 → **审批** | 状态 `approved`，扣减预算（如已配置） |
| 5 | **收货** → 选主仓库 | 库存增加；应付单自动生成 |
| 6 | 应付管理 → **付款** | 状态 `open` → `partial` → `paid` |
| 7 | 财务报表 | 「应付未付」减少；刷新后可见最新汇总 |

![采购订单](images/erp-procurement-order.png)

**权限要求：** `procurement:request`、`procurement:order`、`procurement:receive`、`finance:ap`、`finance:report`。

若补货列表为空，请在 **产品管理** 提高安全库存，或等待库存消耗至阈值以下后再查看。

## 7. 销售管理

### 7.1 业务流程

**订单主线：**

```
销售报价 → 审批 → 转销售订单 → 审批 → 发货（扣库存）→ 应收生成 → 售后（可选）
```

**线索主线（沈阳公海 POI）：**

```
公海浏览/筛选 → 领取至我的线索 → 联系上报（可选上传录音）→ 主管审核 → 转客户 / 继续跟进
```

> **外勤 App** 可完成公海领取（含批量）、我的线索、联系上报、导航到店、消息通知等；报价/订单/发货/统计仍在 **Web ERP** 操作。详见 **[外勤 App 使用说明](FIELD_APP_USER_GUIDE.md)**。

### 7.2 客户管理

维护客户档案、联系人、信用额度。

### 7.3 销售报价

1. 新建报价单，添加产品行与报价单价。
2. 审批后可 **转为销售订单**。

### 7.4 销售订单

1. 创建/转换订单，确认客户与明细。
2. 审批通过后可在 **发货** 中选择仓库出库。
3. 发货前系统 **校验库存**；不足则提示失败。
4. 发货成功后生成 **应收单**（详见 [10.2 应收管理](#102-应收管理)）。

### 7.5 发货管理

对已审批订单执行出库，扣减对应仓库库存。

### 7.6 售后服务

路径：**销售管理 → 售后服务**

- 创建售后工单，关联客户/销售订单。
- 跟踪处理状态（待处理、处理中、已完成等）。

### 7.7 公海线索

路径：**销售管理 → 公海线索**

公海存放尚未分配的销售线索（本系统已导入沈阳市真实 POI 商户数据，含店名、电话、地址、区县、行业分类等）。

![公海线索](images/erp-leads-pool.png)

**页面说明：**

| 元素 | 说明 |
|------|------|
| 已持有 X / 50 条 | 当前用户私海持有量上限为 **50** 条 |
| 区县 / 分类 / 电话 | 按行政区、高德一级行业、是否有电话筛选 |
| 店名/电话/地址 | 关键词模糊搜索 |
| 领取 | 将单条线索划入「我的线索」，进入 **15 天保护期** |
| 批量领取 | 勾选多行后一次性领取（受上限约束） |

**操作步骤：**

1. 使用 `sales_clerk`（销售专员）或具备 `lead:claim` 权限的账号登录。
2. 按需设置筛选项，点击 **查询**。
3. 点击行内 **领取**，或勾选后 **批量领取**。
4. 领取成功后，线索从公海列表消失，出现在 **我的线索**。

> 保护期内的线索仅持有人可跟进；超期未有效跟进将自动回收至公海（系统定时任务）。

### 7.8 我的线索与联系上报

路径：**销售管理 → 我的线索**

![我的线索](images/erp-leads-mine.png)

| 列 | 说明 |
|----|------|
| 保护期 | 剩余天数，到期未跟进将回收到公海 |
| 跟进次数 | 含联系上报与手工跟进记录 |
| 操作 | **联系上报**、跟进、转为客户、释放、标记无效 |

#### 联系上报（结构化跟进）

销售专员完成一次电话/拜访后，应使用 **联系上报** 提交结构化记录（全部进入待审队列，由主管审核）。可在 **Web 我的线索** 或 **外勤 App → 我的线索 → 联系上报** 操作。

![联系上报表单](images/erp-leads-contact-report.png)

**操作步骤（Web / App 字段一致）：**

1. 在「我的线索」点击 **联系上报**（Web 为右侧抽屉，App 为独立页面）。
2. 填写必填项：
   - **跟进方式**：电话 / 拜访 / 微信 / 其他
   - **联系结果**（见下表）
   - **沟通摘要**：客户反馈、意向、下次计划等
3. **电话录音（可选）**：
   - **Web**：点击 **上传录音**，选择本地 mp3/m4a/wav
   - **App**：点击 **选择录音文件**，从手机存储选取（不支持 App 内录音）
   - 单文件 ≤ 10MB，存储于 MinIO
4. **质量标记**（Web 可选）：有效 / 空号 / 已关店等。
5. 点击 **提交上报**。提交后状态为 **待审核**，跟进次数 +1；主管会收到 **消息通知**。

> **App 离线**：无网络时可暂存上报，恢复网络后自动提交（见 [外勤 App 使用说明 §6.4](FIELD_APP_USER_GUIDE.md#64-离线暂存)）。

**联系结果枚举：**

| 结果 | 说明 | 附加要求 |
|------|------|----------|
| 已接通 | 电话接通并完成沟通 | — |
| 未接/挂断 | 无人接听或挂断 | — |
| 有意向 | 客户表达合作意向 | — |
| 明确拒绝 | 客户明确不需要 | — |
| 约下次联系 | 约定下次回访时间 | 必须填写 **下次联系时间** |
| 空号/错号 | 号码无效 | — |
| 已关店 | 商户已停业 | — |

**权限：** `lead:report`（销售专员默认具备）。重新登录后权限才会刷新。

### 7.10 公海数据导入（管理员）

路径：**系统管理 → 线索导入**（`/system/leads/import`，需 `lead:import` 权限，`admin` 默认具备）

![线索导入](images/erp-system-leads-import.png)

公海线索有三种导入方式，按数据量选择：

| 场景 | 推荐方式 |
|------|----------|
| 自有 Excel/CRM 导出，几百～几千条 | **页面上传 CSV** 或粘贴 JSON |
| 外部系统定时同步 | **API** `POST /api/leads/import` |
| 沈阳全城 POI、尽量补全电话 | **运维脚本** `import-shenyang-poi.mjs` + 高德 Key |

导入完成后，在 **销售管理 → 公海线索** 查看数量与筛选项是否正常。

#### 方式 A：页面上传 CSV

1. 使用 `admin` 登录 → **系统管理 → 线索导入**。
2. 点击 **上传 CSV**，选择 UTF-8 编码的 CSV 文件。
3. 导入结束后页面显示 **新增 / 跳过（重复）/ 失败** 数量。

**CSV 表头（第一行）：**

```text
name,phone,address,district,category,source,source_id
```

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 店名/商户名 |
| `phone` | 否 | 主联系电话 |
| `address` | 否 | 详细地址 |
| `district` | 否 | 区县，建议与公海筛选项一致（沈阳 13 区县，见下表） |
| `category` | 否 | 行业大类，建议与公海「分类」筛选项一致（见下表） |
| `source` | 否 | 数据来源，默认 `manual`；有外部 ID 时填 `amap`、`tencent` 等 |
| `source_id` | 否 | 外部 POI 编号，与 `source` 配合用于去重 |

**可选扩展列**（表头可用下划线）：`phone_backup`（备用电话）、`lng`、`lat`（经纬度）。

**CSV 示例：**

```csv
name,phone,address,district,category,source,source_id
云之印图文广告,15840454431,黄河北大街25号,皇姑区,生活服务,manual,
恩林房翻新(浑南店),13889173430,浑南区某路88号,浑南区,生活服务,amap,B0FFXXXX
```

**沈阳 13 区县（`district` 建议值）：**  
和平区、沈河区、大东区、皇姑区、铁西区、苏家屯区、浑南区、沈北新区、于洪区、辽中区、新民市、康平县、法库县。

**行业大类（`category` 建议值，与公海筛选一致）：**  
餐饮服务、购物服务、生活服务、体育休闲服务、医疗保健服务、住宿服务、汽车服务、汽车销售、汽车维修、摩托车服务、科教文化服务、金融保险服务、公司企业、商务住宅、风景名胜、公共设施、其他。

#### 方式 B：页面粘贴 JSON

同一路径下方 **JSON 文本框**，格式示例：

```json
[
  {
    "name": "示例餐饮店",
    "phone": "024-12345678",
    "address": "和平区某某路1号",
    "district": "和平区",
    "category": "餐饮",
    "source": "manual"
  }
]
```

点击 **导入 JSON** 提交。字段与 CSV 相同，亦支持 `phoneBackup`、`lng`、`lat`、`remark` 等。

#### 方式 C：API 批量导入

适合脚本或第三方系统对接：

```http
POST /api/leads/import
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "items": [
    {
      "name": "示例店",
      "phone": "13800001111",
      "district": "铁西区",
      "category": "购物服务"
    }
  ]
}
```

响应示例：`{ "created": 100, "skipped": 5, "failed": 0, "errors": [] }`  
`skipped` 表示与库内已有记录重复而未写入。

#### 方式 D：沈阳全量 POI 脚本（大批量）

已在本地导入约 2.7 万条沈阳真实 POI 时，使用的是项目根目录运维脚本（直连数据库，不经过页面）：

```powershell
cd backend
node ../scripts/import-shenyang-poi.mjs

# 清空现有公海后重新导入
node ../scripts/import-shenyang-poi.mjs --replace

# 仅高德 / 仅 OpenStreetMap
node ../scripts/import-shenyang-poi.mjs --amap-only
node ../scripts/import-shenyang-poi.mjs --osm-only
```

**前置条件：**

- PostgreSQL 可连接，已执行 migrate 与 seed。
- `backend/.env` 中配置 `AMAP_API_KEY`（[高德开放平台](https://lbs.amap.com/) 申请 **Web 服务** Key）。
- 可选：导入后执行 `node scripts/recategorize-leads.mjs` 统一行业分类。

#### 去重规则

系统写入前自动去重，重复记录会计入 **跳过**，不会覆盖已有数据：

| 条件 | 去重键 |
|------|--------|
| 填写了 `source` + `source_id`，且 `source` 不是 `manual` | 店名 + 来源 + 外部 ID |
| 手工导入或未填外部 ID | 店名 + 电话 + 区县 |

#### 导入后验证

1. **销售管理 → 公海线索**：列表有数据，区县/分类筛选可用。
2. **销售管理 → 线索统计**（如有权限）：查看公海/已领取数量分布。
3. 用 `sales_clerk` 试领 1 条，确认 **我的线索** 与 **联系上报** 流程正常。

## 8. 库存管理

### 8.0 业务总览

库存模块是采购、销售、生产的 **数量中枢**，核心链路如下：

```
主数据（产品/仓库）→ 台账 → 出入库/调拨/盘点 → 预警与追溯
     ↑ 采购收货 / 销售发货 / 生产领料·完工 等业务自动写入流水
```

| 子模块 | 菜单路径 | 所需权限 | 说明 |
|--------|----------|----------|------|
| 库存台账 | 库存管理 → 库存台账 | `inventory:stock` | 各仓当前数量 |
| 出入库管理 | 库存管理 → 出入库管理 | `inventory:inout` | 手工调整与流水 |
| 库存调拨 | 库存管理 → 库存调拨 | `inventory:inout` | 仓间转移 |
| 库存盘点 | 库存管理 → 库存盘点 | `inventory:inout` | 账实核对 |
| 库存预警 | 库存管理 → 库存预警 | `inventory:alert` | 低于安全库存 |
| 批次追溯 | 库存管理 → 批次追溯 | `inventory:trace` | 批次/序列号 |
| 产品管理 | 库存管理 → 产品管理 | `inventory:stock` | 含安全库存 |
| 仓库管理 | 库存管理 → 仓库管理 | `inventory:stock` | 仓库主数据 |

演示环境 seed 预置 **主仓库 WH001**、产品 **P001（原材料）**、**FG001（成品）** 及初始库存。

---

### 8.1 库存台账

路径：**库存管理 → 库存台账**

![库存台账](images/erp-inventory-stock.png)

- 按 **产品 × 仓库 × 批次号** 展示当前库存；无批次时批次列为空。
- 同一产品在不同仓库、不同批次会 **分多行** 显示（如主仓/副仓、批次库存）。
- 点击 **+ 录入库存** 可手工初始化某仓库存记录。
- 列表支持 **删除** 空库存行（谨慎操作）。

**业务自动更新：** 采购收货、销售发货、生产领料/完工、调拨、盘点差异等都会反映在此台账。

---

### 8.2 出入库管理

路径：**库存管理 → 出入库管理**

![出入库流水](images/erp-inventory-inout.png)

记录所有库存变动流水，常见类型：

| 类型 | 来源 |
|------|------|
| `purchase_in` | 采购收货 |
| `sales_out` | 销售发货 |
| `production_out` / `production_in` | 工单开工 / 完工 |
| `transfer_in` / `transfer_out` | 库存调拨 |
| `adjustment` | 手工库存调整 |
| `stocktake` | 盘点差异调整 |

#### 手工库存调整

![库存调整弹窗](images/erp-inventory-inout-modal.png)

1. 点击 **库存调整**。
2. 选择类型（调整）、产品、仓库、**数量**（正数入库、负数出库）。
3. 可选填 **批次号**、参考单号。
4. 确认后生成流水并更新台账。

> 若调整后库存为负，系统返回 **「库存不足」** 并拒绝。

---

### 8.3 库存调拨

路径：**库存管理 → 库存调拨**

![库存调拨](images/erp-inventory-transfer.png)

1. 选择 **产品**、**源仓库**、**目标仓库**、数量。
2. 可选填参考单号。
3. 点击 **确认调拨**。

系统在同一事务中生成 **transfer_out**（源仓减）与 **transfer_in**（目标仓增）两条流水。**源仓与目标仓不能相同**。

> 调拨前请确认源仓库库存充足；需至少 **两个仓库**（可在仓库管理中新增，如 WH002 副仓库）。

---

### 8.4 库存盘点

路径：**库存管理 → 库存盘点**

![库存盘点](images/erp-inventory-stocktake.png)

1. 点击 **+ 新建盘点**，选择 **仓库**（该仓须有库存记录）。
2. 系统按当前账面数量生成盘点单（状态 **盘点中 counting**）及明细行。
3. 点击 **明细**，录入各产品 **实盘数量**。
4. 全部明细录入后，点击 **完成盘点**。

![盘点明细](images/erp-inventory-stocktake-detail.png)

- 完成时按 **实盘 − 账面** 差异自动生成 **stocktake** 类型出入库调整。
- 若有明细未录入实盘数量，完成将被拒绝。
- **已完成** 盘点单不可删除。

---

### 8.5 库存预警

路径：**库存管理 → 库存预警**

![库存预警](images/erp-inventory-alert.png)

列出 **全仓合计库存 < 安全库存** 的产品，显示当前库存、安全库存与 **缺口**。

- 安全库存在 **产品管理** 中维护（字段 **安全库存**，须 > 0 才参与预警）。
- 采购收货、生产完工、手工入库等使库存回升后，预警行会自动消失。
- 与 **报表中心 → 智能补货** 联动：低于安全库存的产品会出现补货建议。

---

### 8.6 产品与仓库主数据

#### 产品管理

路径：**库存管理 → 产品管理**

![产品管理](images/erp-inventory-product.png)

维护产品编码、名称、分类、单位、单价、**安全库存**、是否序列号管理等。安全库存是预警与智能补货的基础。

#### 仓库管理

路径：**库存管理 → 仓库管理**

![仓库管理](images/erp-inventory-warehouse.png)

维护仓库编码、名称、地址。调拨、盘点、业务单据收货/发货均依赖仓库主数据。

---

### 8.7 批次 / 序列号追溯

路径：**库存管理 → 批次追溯**

#### 批次追溯 Tab

![批次追溯](images/erp-inventory-trace-batch.png)

1. 输入 **批次号**（出入库调整时可指定 batchNo）。
2. 点击 **查询**，展示该批次当前库存分布与全部流转流水。

#### 序列号追溯 Tab

![序列号追溯](images/erp-inventory-trace-serial.png)

1. 切换到 **序列号追溯**，输入序列号查询生命周期流水。
2. 或通过 **登记序列号** 为新序列号建立档案（关联产品、批次、仓库）。

#### 区块链存证（可选）

- 页面顶部显示 RPC 配置状态；未配置时为 **本地哈希链**。
- 支持 **上链存证 / 链上验真**（需配置 `BLOCKCHAIN_RPC_URL` 与 `BLOCKCHAIN_PRIVATE_KEY`）。

---

### 8.8 实操演练（推荐顺序）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 库存台账 → 查看 FG001 / P001 | 见主仓库数量 |
| 2 | 出入库管理 → 库存调整 +10 | 台账增加；流水出现 adjustment |
| 3 | 仓库管理 → 新增副仓库 | 调拨可用两个仓库 |
| 4 | 库存调拨 → 主仓转副仓 5 件 | 两仓数量变化；transfer 流水 |
| 5 | 库存盘点 → 新建 → 录入实盘 → 完成 | 差异自动 adjustment |
| 6 | 产品管理 → 提高安全库存 | 库存预警出现该产品 |
| 7 | 批次追溯 → 输入批次号查询 | 见库存与流水 |

---

### 8.9 与其他模块联动

| 业务 | 库存影响 | 流水类型 |
|------|----------|----------|
| 采购收货 | 原材料 **增加** | `purchase_in` |
| 销售发货 | 成品 **减少** | `sales_out` |
| 工单开工 | 组件 **减少** | `production_out` |
| 工单完工 | 成品 **增加** | `production_in` |
| 智能补货 | 预警驱动采购 → 最终收货回补 | 间接 |

发货/开工提示 **库存不足** 时，应先在台账确认数量，或通过采购收货、生产完工、手工调整补充。

## 9. 生产管理

### 9.0 业务总览

生产模块与 **库存** 联动，典型链路如下：

```
BOM 维护 → 生产计划 → 审批 → 工单 → 下达 → 开工(扣减组件库存) → 完工入库(增加成品库存) → 质检
```

| 子模块 | 菜单路径 | 所需权限 | 说明 |
|--------|----------|----------|------|
| BOM 管理 | 生产管理 → BOM 管理 | `production:bom` | 成品与组件用量 |
| 生产计划 | 生产管理 → 生产计划 | `production:plan` | 计划数量与日期 |
| 工单管理 | 生产管理 → 工单管理 | `production:workorder` | 下达、开工、完工 |
| 质量管理 | 生产管理 → 质量管理 | `production:quality` | 工单质检记录 |

演示环境 seed 预置 **BOM-FG001**（成品 FG001，每件消耗原材料 P001 × 2）及主仓库库存，可直接演练完整流程。

---

### 9.1 BOM 管理

路径：**生产管理 → BOM 管理**

![BOM 管理](images/erp-production-bom.png)

1. 点击 **+ 新建 BOM**，填写 BOM 编码、名称、**成品**、版本（默认 1.0）。
2. 添加 **组件行**：选择组件产品、单位用量、计量单位；至少一行，否则保存失败。
3. 列表中点击 **展开行** 可查看组件明细。

![BOM 组件明细](images/erp-production-bom-detail.png)

| 字段 | 说明 |
|------|------|
| 成品 | 工单完工入库的产品（如 FG001 成品组件X） |
| 组件 / 用量 | 生产 1 件成品所需组件数量；工单创建时按 **计划数量 × 用量** 生成领料需求 |

演示 BOM：**BOM-FG001**，成品 FG001，组件 P001 用量 **2 件/成品**。

---

### 9.2 生产计划

路径：**生产管理 → 生产计划**

![生产计划](images/erp-production-plan.png)

1. 点击 **+ 新建计划**，填写名称、产品、计划数量、开始/结束日期。
2. 保存后状态为 **草稿（draft）**，可删除或修改。
3. 点击 **审批（✓）** 后状态变为 **已审批（approved）**；已审批计划可在创建工单时关联。

> 仅 **草稿** 计划可编辑/删除；已审批计划用于指导工单，不会自动下达工单。

---

### 9.3 工单管理

路径：**生产管理 → 工单管理**

![工单管理](images/erp-production-work-order.png)

#### 状态流转

```
draft（草稿）→ released（已下达）→ in_progress（进行中）→ completed（已完成）
```

| 操作 | 当前状态 | 说明 |
|------|----------|------|
| **下达** | 草稿 | 确认排产，记录开始日期 |
| **开工** | 已下达 | 按 BOM 扣减 **组件库存**（`production_out`）；库存不足则失败 |
| **完工** | 进行中 / 已下达 | 成品入指定仓库（`production_in`），完成数量 = 计划数量 |
| **删除** | 草稿 | 已下达及以后不可删 |

#### 新建工单

1. 点击 **+ 新建工单**，选择 **产品**、可选 **BOM**、可选 **已审批生产计划**、计划数量。
2. 若选择 BOM，系统自动生成 **领料明细**（组件 × 计划数量）。
3. 按上表依次 **下达 → 开工 → 完工**。

![完工入库](images/erp-production-work-order-complete.png)

完工时选择 **入库仓库**（通常为主仓库），确认后成品库存增加、工单状态变为 **已完成**。展开工单行可查看领料与已消耗数量。

**库存校验：** 开工前请确认 **库存台账** 中组件（如 P001）数量充足；不足时提示「物料 xxx 库存不足」。

---

### 9.4 质量管理

路径：**生产管理 → 质量管理**

![质量管理](images/erp-production-quality.png)

1. 点击 **+ 新建质检单**，选择 **进行中或已完工** 的工单。
2. 填写 **检验数量**、**合格数**、**不合格数**、检验说明。
3. **合格数 + 不合格数** 必须等于 **检验数量**，否则提交失败。

![新建质检单](images/erp-production-quality-form.png)

### 9.6 与库存的关系

| 环节 | 库存影响 | 流水类型 |
|------|----------|----------|
| 工单开工 | 组件仓库 **减少** | `production_out` |
| 工单完工 | 成品仓库 **增加** | `production_in` |

组件库存不足时无法开工；可通过 **采购收货** 或 **库存出入库** 补充原材料后再试。

## 10. 财务管理

### 10.0 业务总览

财务模块与采购、销售联动，核心链路如下：

```
销售：报价 → 销售订单 → 审批 → 发货（扣库存）→ 自动生成应收单 → 登记收款
采购：申请 → 采购订单 → 审批 → 收货（增库存）→ 自动生成应付单 → 登记付款
总账：维护科目 → 录入凭证 → 过账 → 财务报表科目余额表更新
```

| 子模块 | 菜单路径 | 所需权限 | 说明 |
|--------|----------|----------|------|
| 总账管理 | 财务管理 → 总账管理 | `finance:gl` | 科目与凭证 |
| 应收管理 | 财务管理 → 应收管理 | `finance:ar` | 销售发货后自动生成 |
| 应付管理 | 财务管理 → 应付管理 | `finance:ap` | 采购收货后自动生成 |
| 固定资产 | 财务管理 → 固定资产 | `finance:asset` | 资产卡片与折旧 |
| 预算管理 | 财务管理 → 预算管理 | `finance:budget` | 采购审批扣减预算 |
| 财务报表 | 财务管理 → 财务报表 | `finance:report` 或 `report:center` | 汇总与科目余额 |

演示环境 seed 会预置 **7 个会计科目**、**1 条固定资产**、**1 条年度预算**，以及若干已审批的采购/销售订单，便于直接演练收付款流程。

### 10.2 应收管理

路径：**财务管理 → 应收管理**

![应收管理](images/erp-finance-ar.png)

#### 自动生成规则

- 触发条件：**销售订单审批通过** 后，在 **发货管理** 或销售订单页执行 **发货**。
- 系统校验对应仓库 **库存充足**，出库成功后自动生成 **应收单**（单据号形如 `AR-YYYYMMDD-0001`）。
- 应收金额 = 销售订单总金额；到期日默认为生成日后 **30 天**。
- 同一销售订单 **不会重复** 生成应收单。

前置操作详见 [7.4 销售订单](#74-销售订单)、[7.5 发货管理](#75-发货管理)。

#### 登记收款

![登记收款](images/erp-finance-ar-receipt.png)

1. 在应收列表找到状态为 **未结清** 的单据（`open` / `partial`）。
2. 点击 **收款**，输入本次收款金额（默认为剩余未收金额，可改）。
3. 点击 **确认** 后更新 **已收金额** 与状态：

| 状态 | 含义 |
|------|------|
| `open` | 未收款 |
| `partial` | 部分收款 |
| `paid` | 已全部收讫（不再显示收款按钮） |

> 单次收款金额不得超过 **应收余额**（应收金额 − 已收金额）。

### 10.3 应付管理

路径：**财务管理 → 应付管理**

![应付管理](images/erp-finance-ap.png)

#### 自动生成规则

- 触发条件：**采购订单审批通过** 后，在 **收货验收** 页完成 **收货**。
- 系统按订单明细 **增加库存**，并自动生成 **应付单**（单据号形如 `AP-YYYYMMDD-0001`）。
- 应付金额 = 采购订单总金额；到期日默认为生成日后 **30 天**。
- 同一采购订单 **不会重复** 生成应付单。

前置操作详见 [6.4 采购订单](#64-采购订单)、[6.5 收货验收](#65-收货验收)。

#### 登记付款

![登记付款](images/erp-finance-ap-payment.png)

1. 在应付列表找到状态为 **未结清** 的单据（`open` / `partial`）。
2. 点击 **付款**，输入本次付款金额（默认为剩余未付金额，可改）。
3. 点击 **确认** 后更新 **已付金额** 与状态：

| 状态 | 含义 |
|------|------|
| `open` | 未付款 |
| `partial` | 部分付款 |
| `paid` | 已全部付讫（不再显示付款按钮） |

> 单次付款金额不得超过 **应付余额**（应付金额 − 已付金额）。

### 10.4 固定资产

路径：**财务管理 → 固定资产**

![固定资产](images/erp-finance-assets.png)

1. 点击 **+ 新增资产**，填写资产编号、名称、分类、原值、使用年限（月）、购置日期、存放地点等。
2. 对 **在用（active）** 资产可执行：
   - **计提折旧**：按直线法增加累计折旧；
   - **处置**：标记为已处置，不再参与净值统计。
3. 已处置资产可删除卡片。

演示数据含一条 **办公服务器**（原值 60,000）。计提折旧、处置或新增资产后，**财务报表** 与 **智能财务** 中的「固定资产净值」会 **立即更新**（与应收/应付一样自动清除报表缓存）。

![固定资产 — 计提折旧后](images/erp-finance-assets-active.png)

### 10.5 预算管理

路径：**财务管理 → 预算管理**

![预算管理](images/erp-finance-budget.png)

1. 点击 **+ 新增预算**，设置编码、名称、年度、分类（如 `procurement` 采购）、部门、预算总额。
2. **采购订单审批** 时，系统按订单金额 **扣减** 对应年度、分类下 **在用（active）** 预算的已用额度。
3. 若 **已用 + 本次订单金额 > 预算总额**，审批 **被拒绝**（HTTP 400），订单保持 **草稿**；需先 **追加预算总额**（编辑预算）或 **调减订单金额** 后再审批。
4. 审批通过后若仍触发超支（如并发审批），系统会发送 **预算超支** 站内通知。

![预算管理 — 使用率](images/erp-finance-budget-usage.png)

**示例：** 2026 年采购预算总额 500,000，已用 162,000；若新建订单金额 350,000，审批将提示「超出预算可用额度」。

### 10.6 财务报表

路径：**财务管理 → 财务报表**

![财务报表](images/erp-finance-report.png)

页面分为 **汇总卡片** 与 **科目余额表** 两部分。

#### 汇总指标

| 指标 | 说明 |
|------|------|
| 应收未收 | 所有应收单的（应收金额 − 已收金额）合计 |
| 应付未付 | 所有应付单的（应付金额 − 已付金额）合计 |
| 固定资产净值 | 在用资产原值 − 累计折旧 |
| 在用资产数 | 状态为 `active` 的资产条数 |

#### 科目余额表

- 数据来源：**已过账** 的总账凭证分录，按科目汇总借方、贷方与余额。
- 无过账凭证时，该表可能为空或仅显示历史过账数据。

#### 数据刷新

- **应收/应付收付款**、**凭证过账**、**发货/收货生成应收应付**、**固定资产变更（新增/折旧/处置）** 后，报表汇总会 **立即更新**（服务端自动清除财务报表缓存）。
- 若页面仍显示旧数据，**刷新浏览器** 即可；报表中心 **运营概览** 仍可能有约 60 秒缓存。

---

#### 13.2.1 智能补货 Tab

![智能补货列表](images/erp-intelligence-replenishment.png)

**出现条件：** 与 **库存预警** 相同——产品 **全仓合计库存 < 安全库存**（安全库存须 > 0）。

**建议算法：**

```
日均消耗 = 近30天出库量(sales_out/transfer_out/负adjustment) ÷ 30
lead time 需求 = ceil(日均消耗 × 7)
建议采购量 = max(缺口, lead time 需求, 1)
可覆盖天数 = 当前库存 ÷ 日均消耗（无出库时为 null）
```

![智能补货字段明细](images/erp-intelligence-replenishment-detail.png)

| 字段 | 说明 |
|------|------|
| 当前库存 / 安全库存 / 缺口 | 与库存预警一致 |
| 建议采购量 | 上式计算结果 |
| 日均消耗 | 近 30 天出库 ÷ 30 |
| 可覆盖天数 | 按当前库存与日均消耗估算 |
| 优先级 | **高**：库存 ≤ 50% 安全库存；**低**：缺口 ≤ 20% 安全库存；否则 **中** |
| 原因 | `velocity_and_safety`（有出库速度）或 `safety_stock_only` |

**操作：**

| 按钮 | 说明 |
|------|------|
| **一键全部转采购申请** | 为当前所有建议行生成 **一张** 采购申请（标题「智能补货采购申请」） |
| **转采购申请**（行内） | 仅为该产品生成采购申请，数量 = 建议采购量 |

> 若无低于安全库存的产品，列表为空且「一键全部」按钮禁用。  
> 全部产品安全库存设为 0 时，转申请 API 返回 400「没有可生成采购申请的产品」。

完整采购闭环见 [6.6 智能补货全流程](#66-智能补货全流程)。

---

#### 13.2.2 智能财务 Tab

![智能财务洞察](images/erp-intelligence-finance.png)

调用 `GET /api/intelligence/finance`，基于 [10.6 财务报表](#106-财务报表) 与运营概览 **实时** 生成洞察卡片（severity：info / warning / success）：

| 洞察 code | 含义 | 典型 severity |
|-----------|------|---------------|
| `ar_outstanding` | 应收未收金额与未结清笔数 | 应收 > 应付时为 warning |
| `ap_outstanding` | 应付未付金额与未结清笔数 | 应付 > 应收时为 warning |
| `net_cash_position` | 净头寸 = 应收未收 − 应付未付 | ≥0 success，<0 warning |
| `fixed_asset_net` | 在用固定资产净值与数量 | info |
| `gross_margin_hint` | 销售总额 vs 采购总额、毛利率 | 毛利率 <15% warning |
| `balance_sheet_snapshot` | 资产 / 负债 / 权益科目余额汇总 | info |

> **与财务报表区别：** 财务管理 → 财务报表展示 **明细表格**（科目余额表等）；智能财务 Tab 展示 **规则摘要卡片**，便于管理层快速浏览。  
> 收付款、计提折旧后数据 **立即** 更新（同 `report:finance` 缓存失效机制）。

| 4 | 采购申请 → 查看明细 | 数量 ≥ 建议采购量 |

2. 填写 **租户编码**（唯一，如 `acme`）与 **租户名称**。
3. 保存后，用户登录时在租户字段输入该编码即可进入隔离数据空间。

> 默认租户编码为 **`default`**。租户编码 **不可重复**，重复提交将报错。新租户需另行 seed 或创建管理员用户。

## 17. 常见问题

### Q1：登录提示用户名或密码错误？

- 确认租户为 `default`。
- 确认密码与 `backend/.env` 中 seed 配置一致。
- 检查后端是否运行、数据库是否已 seed。

### Q2：菜单项缺失？

- 当前角色无对应权限；联系管理员在 **角色管理** 中分配。
- 重新登录使权限生效。

### Q3：访问页面显示 403？

- 直接输入了无权限的路由；从侧栏进入有权限的页面即可。

### Q4：发货提示库存不足？

- 在 **库存台账** 检查对应仓库数量。
- 通过采购收货或生产完工补充库存。

### Q5：库存预警无数据？

- 在 **产品管理** 设置 **安全库存 > 0**，且当前库存低于该值。

### Q6：报表 / BI 数据不是最新？

- **财务报表**（财务管理 → 财务报表）：发货/收货、收付款、凭证过账后服务端会 **立即清除缓存**；若页面数字未变，请 **刷新浏览器**。
- **报表中心运营概览**、**BI 大屏**：仍有约 **60 秒** 缓存，稍等或强刷即可。
- 生产环境配置 Redis 后，多实例共享缓存；财务业务操作会自动失效 `report:finance` 缓存项。

### Q7：权限保存后未生效？

- 用户需 **退出并重新登录**。
- 确认保存时无报错（权限列表接口正常）。

### Q8：区块链上链失败？

- 未配置 RPC 时仅本地哈希链，属正常。
- 配置 `BLOCKCHAIN_RPC_URL` 与 `BLOCKCHAIN_PRIVATE_KEY` 后重试。

### Q9：联系上报按钮灰色或提交失败？

- 确认当前账号持有该线索（**我的线索** 中操作，非公海）。
- 销售专员需具备 `lead:report` 权限；修改权限后 **重新登录**。
- 选择「约下次联系」时必须填写 **下次联系时间**。

### Q10：上报审核页无数据或显示 403？

- **403**：当前角色无 `lead:review`；销售专员只能上报，不能审核。
- **无数据**：待审核为空表示已全部处理；可切换 **全部上报** 查看历史。
- 页面首次加载若 token 未就绪，**刷新** 即可。

### Q11：录音上传或播放失败？

- 确认 MinIO 已启动（Docker：`docker compose` 中 `minio` 服务）。
- 检查 `backend/.env` 中 `MINIO_ENDPOINT`、`MINIO_ACCESS_KEY`、`MINIO_SECRET_KEY`。
- 仅支持音频格式，单文件 ≤ 10MB。

### Q12：公海领取提示已达上限？

- 单人最多同时持有 **50** 条线索；请先 **释放** 或 **转为客户** 后再领取。

### Q13：如何向公海批量导入线索？

- 小批量：管理员登录 → **系统管理 → 线索导入**，上传 CSV 或粘贴 JSON（详见 [7.10 公海数据导入](#710-公海数据导入管理员)）。
- 大批量（沈阳 POI）：配置 `AMAP_API_KEY` 后运行 `scripts/import-shenyang-poi.mjs`。
- 接口对接：调用 `POST /api/leads/import`。
- 重复数据会自动 **跳过**，不会覆盖已有记录。

> **外勤 App** 安装、登录、批量领取、离线上报等问题，见 **[外勤 App 使用说明 — 常见问题](FIELD_APP_USER_GUIDE.md#13-常见问题)**。

### Q14：应收或应付列表为空？

- **应收**：须 **销售订单审批 + 发货** 后才会自动生成（见 [10.2](#102-应收管理)）；仅创建订单不够。
- **应付**：须 **采购订单审批 + 收货验收** 后才会自动生成（见 [10.3](#103-应付管理)）。
- 确认账号具备 `finance:ar` / `finance:ap` 权限。

### Q15：收款或付款提示金额超限？

- 单次金额不能超过 **未收/未付余额**（总额 − 已收/已付）。
- 状态为 `paid` 的单据已结清，不再显示收款/付款按钮。

### Q16：总账凭证无法过账？

- 检查借贷是否 **平衡**（借方合计 = 贷方合计）。
- 仅 **草稿（draft）** 凭证可过账；已过账凭证不可删除，需通过反向凭证调整。

### Q17：登录页提示「服务器错误」？

- 确认 **Docker Desktop 已启动**（PostgreSQL 在 5432 端口）。
- 确认 **后端已运行**（开发环境默认 `http://localhost:3001`）。
- 若 Git 推送 GitHub 失败，本机需配置代理，例如：  
  `git -c http.proxy=http://127.0.0.1:7897 push origin master`（端口以实际为准）。

### Q18：智能补货列表为空？

- 在 **产品管理** 为产品设置 **安全库存**，且 **当前库存低于安全库存** 才会出现建议行。
- 可在 **库存预警** 页先确认有哪些产品低于阈值。
- 转采购申请后，补货建议可能因库存/阈值变化而消失，属正常现象。

### Q19：付款后财务报表数字未变？

- 服务端已自动清除 `report:finance` 缓存；**刷新浏览器** 即可。
- 确认操作的是 **未结清** 应付/应收单；`paid` 状态单据不可再次收付款。

### Q20：采购订单审批提示超出预算？

- 系统按 **当前年度 + 分类 procurement** 的 **在用预算** 校验；**已用 + 订单金额** 不得超过 **预算总额**。
- 可在 **预算管理** 中查看已用/使用率，或 **调减订单** / **提高预算总额** 后重试。
- 未配置对应预算时，审批 **不做预算拦截**（仅在有匹配预算时生效）。

### Q21：计提折旧后报表净值未变？

- 修复后折旧/处置/新增资产会自动刷新报表缓存；若仍异常请 **刷新页面** 并确认后端已重启到最新版本。

### Q22：工单开工提示物料库存不足？

- 在 **库存台账** 查看 BOM 组件（如 P001）是否足够；工单按 **计划数量 × BOM 用量** 扣料。
- 可先 **采购收货** 补充原材料，或减小工单计划数量后再 **开工**。

### Q23：质检提交失败？

- **合格数 + 不合格数** 必须等于 **检验数量**。
- 只能对 **进行中（in_progress）** 或 **已完成（completed）** 的工单创建质检单。

### Q24：库存调整提示「库存不足」？

- 负数量出库会使库存低于 0，系统会拒绝。
- 先在 **库存台账** 确认该仓可用数量，或减小出库数量。

### Q25：盘点无法完成？

- 每条明细须录入 **实盘数量** 后才能完成。
- 盘点单状态为 **已完成** 后不可删除；新建盘点前请确认仓库内有库存记录。

### Q26：调拨失败？

- **源仓库与目标仓库不能相同**；需至少两个仓库（可在 **仓库管理** 新增）。
- 源仓库存不足时调拨会被拒绝，请先在台账确认数量。

### Q27：看不到人力资源菜单？

- 需 `hr:employee` / `hr:attendance` / `hr:salary` / `hr:performance` 中至少一项权限。
- 演示可用 **`hr_clerk`** 登录；`employee` 普通员工账号无 HR 权限。

### Q28：添加工号提示重复？

- **工号全局唯一**，请更换工号后再保存。

### Q29：薪资实发金额不对？

- 实发 = **基本工资 + 奖金 − 扣款**；奖金、扣款留空时按 0 计算。

### Q30：看不到项目管理菜单？

- 需 **`project:manage`**（完整操作）或 **`report:center`**（只读列表）。
- 写操作请使用 **admin**；生产经理等仅有只读权限。

### Q31：项目无法启动？

- 仅 **规划中（planning）** 状态可点 **启动**；已完成项目需新建或调整状态。

### Q32：新建项目提示编码重复？

- **项目编码全局唯一**，请更换编码后再保存。

### Q33：看不到报表中心菜单？

- 需 **`report:center`** 权限；演示可用 **`employee`** 登录。
- **`procurement_clerk`** 等无此权限的角色侧栏不显示报表中心。

### Q34：智能补货列表为空？

- 在 **产品管理** 设置 **安全库存 > 0**，且 **当前库存低于安全库存**。
- 可先查看 **库存预警** 确认有哪些产品低于阈值。

### Q35：转采购申请提示「没有可生成采购申请的产品」？

- 当前 **无任何产品** 低于安全库存，或安全库存均为 0。
- 先在 **库存预警** 或 **智能补货** 确认有建议行后再操作。

### Q36：智能财务卡片数量少或为空？

- 无未结清应收/应付时，对应卡片不出现；**净头寸** 卡片始终展示。
- 若无已过账凭证，资产负债快照可能仅显示部分科目。
