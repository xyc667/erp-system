## 1. 系统概述

本 ERP 系统覆盖企业核心业务流程：

| 模块 | 主要能力 |
|------|----------|
| 采购 | 供应商、采购申请、采购订单、收货验收 |
| 销售 | 客户、报价、销售订单、发货、售后工单、公海线索、联系上报与审核 |
| 库存 | 台账、出入库、调拨、盘点、预警、批次/序列号追溯 |
| 生产 | BOM、生产计划、工单、质检 |
| 财务 | 总账、应收/应付、固定资产、预算、财务报表 |
| 人力 | 员工、部门、岗位、考勤、薪资、绩效 |
| 项目 | 项目与任务进度 |
| 报表 | 运营概览、BI 大屏、智能补货/财务分析 |
| 系统 | 用户、角色、权限、租户、审计、文件、集成 |

技术栈：React 前端 + NestJS 后端 + PostgreSQL，支持 Redis 缓存、MinIO 文件存储、WebSocket 通知。

## 2. 访问与登录

### 2.1 访问地址

| 环境 | 前端地址 | 后端 API |
|------|----------|----------|
| 开发模式 | http://localhost:5173 | http://localhost:3001/api（或见 `backend/.env` 中 `PORT`） |
| Docker Compose | http://localhost:8080 | http://localhost:3000/api |

### 2.2 登录步骤

1. 打开浏览器访问上述前端地址，未登录时会自动跳转到登录页。
2. 填写 **租户**（默认选 `default` / 默认租户）。
3. 填写 **用户名** 和 **密码**。
4. 点击 **登录**。

登录页右上角可切换 **中文 / English**。

### 2.3 默认与演示账号

执行数据库 seed 后会自动创建管理员与多角色演示用户。密码在 `backend/.env` 中通过以下变量配置（**切勿使用默认值部署到公网**）：

| 变量 | 说明 |
|------|------|
| `SEED_ADMIN_PASSWORD` | 管理员 `admin` 的初始密码 |
| `SEED_DEMO_PASSWORD` | 各角色演示用户的统一初始密码 |

| 账号 | 角色 | 用途 |
|------|------|------|
| `admin` | 系统管理员 | 全功能管理 |
| `employee` | 普通员工 | 权限测试（采购申请、库存只读、报表） |
| `finance_director` | 财务总监 | 财务全盘 |
| `finance_clerk` | 财务专员 | 日常记账 |
| `procurement_director` | 采购总监 | 采购全流程 |
| `procurement_clerk` | 采购专员 | 采购执行 |
| `sales_director` | 销售总监 | 销售全盘、上报审核 |
| `sales_clerk` | 销售专员 | 报价/订单/发货、公海领取、联系上报 |
| `production_manager` | 生产经理 | 生产+库存协调 |
| `production_clerk` | 生产专员 | 工单执行 |
| `warehouse_admin` | 仓库管理员 | 库存与追溯 |
| `hr_clerk` | 人事专员 | 人事模块 |

> seed 完成后控制台会列出已创建的用户名；密码以 `.env` 配置为准。

## 3. 界面布局与通用操作

登录成功后进入主界面，由四部分组成：

![数据看板](images/erp-dashboard.png)

### 3.1 左侧导航栏（Sider）

- 按业务域分组：财务、采购、销售、生产、库存、人力等。
- **仅显示当前用户有权限的菜单**；无权限的模块不会出现。
- 窄屏（手机/平板）下自动变为 **Drawer 抽屉菜单**，点击左上角图标展开。

### 3.2 顶栏（Header）

| 元素 | 说明 |
|------|------|
| 面包屑 | 显示当前页面路径，如 `首页 / 采购管理 / 采购申请` |
| 区域设置 | `Shanghai · CNY` 按钮，切换时区与币种（会同步到用户 Profile） |
| 语言 | 中文 / English |
| 通知铃铛 | 站内通知列表，支持 WebSocket 实时推送 |
| 用户名 | 当前登录用户姓名，点击可退出登录 |

### 3.3 主内容区

- 页面标题由 **PageTitle** 组件统一展示。
- 列表页普遍采用 **ResponsiveTable**：宽屏表格、窄屏自动卡片化，便于手机浏览。
- 常见按钮：**新建**、**编辑**、**删除**、**审批**、**刷新** 等，操作结果以右上角 Message 提示。

### 3.4 通用表格操作

1. 点击 **新建** 打开表单弹窗（Modal）。
2. 填写必填项（带红色星号），点击 **保存/创建**。
3. 行内 **编辑** 修改记录；**删除** 需二次确认。
4. 带审批流的单据需先 **提交/审批** 才能进入下一环节。

### 3.5 退出登录

点击 Header 右侧用户名 → **退出登录**。Token 清除后需重新登录。

## 4. 角色、账号与权限

### 4.1 权限模型

- 系统预置 **41 项权限**，按模块分组（system、finance、procurement、sales 等）。
- 每个 **角色** 绑定若干权限；用户通过 **角色** 获得权限。
- API 与菜单均受权限控制：
  - **菜单**：侧栏隐藏无权限项。
  - **路由**：直接访问无权限 URL 显示 403 页面。
  - **接口**：无权限返回 HTTP 403。

### 4.2 预置角色（12 个）

| 角色 | 权限数 | 典型使用人 |
|------|--------|------------|
| 系统管理员 | 41（全部） | IT / 实施顾问 |
| 财务总监 | 7 | 财务负责人 |
| 财务专员 | 5 | 会计 |
| 采购总监 | 7 | 采购经理 |
| 采购专员 | 4 | 采购员 |
| 销售总监 | 9+ | 销售经理（含线索审核） |
| 销售专员 | 8+ | 销售员（含公海与上报） |
| 生产经理 | 8 | 车间主任 |
| 生产专员 | 4 | 计划员/操作工 |
| 仓库管理员 | 5 | 仓管 |
| 人事专员 | 5 | HR |
| 普通员工 | 4 | 一线员工 |

### 4.3 配置角色权限

路径：**系统管理 → 角色管理**

![角色管理](images/erp-role-management.png)

**操作步骤：**

1. 在列表中找到目标角色，点击 **权限** 按钮。
2. 在弹窗中按模块勾选权限（finance、inventory、sales 等分组展示）。
3. 点击 **确定** 保存；系统会替换该角色的全部权限绑定。
4. 已登录用户需 **重新登录** 后权限才会刷新（JWT 内不含实时权限变更）。

**新建角色：**

1. 点击 **+ 添加角色**。
2. 填写角色名称、描述 → 创建。
3. 再点击 **权限** 分配具体权限。

### 4.4 用户管理

路径：**系统管理 → 用户管理**

1. **新建用户**：用户名、姓名、邮箱、角色、初始密码。
2. **编辑**：可修改角色、状态（启用/禁用）。
3. 用户被禁用后无法登录。

### 4.5 权限测试建议

用 `employee` 账号（演示密码见 `.env` 中 `SEED_DEMO_PASSWORD`）登录验证：

- 侧栏应仅有：采购申请、部分库存、报表中心等。
- 访问 `/system/role` 应显示 403。
- 可创建采购申请；**不能** 执行出入库调整（403）。

运行自动化审计（可选）：

```powershell
node tests/permissions-audit.mjs
```

## 6. 采购管理

### 6.1 业务流程总览

```
采购申请 → 审批 → 转采购订单 → 审批采购单 → 收货入库 → 应付生成
```

### 6.2 供应商管理

路径：**采购管理 → 供应商管理**

- 维护供应商编码、名称、联系人、电话。
- 采购订单创建时需选择供应商。

### 6.3 采购申请

路径：**采购管理 → 采购申请**

![采购申请](images/erp-procurement.png)

**新建申请：**

1. 点击 **+ 新建申请**。
2. 填写标题、申请原因。
3. 添加明细行：选择产品、填写数量、预估单价（可选）。
4. 保存后状态为 **草稿**，可编辑。
5. 点击 **提交审批** → 审批通过后可用 **转采购单** 生成采购订单。

**智能补货转申请：**

在 **报表中心 → 智能分析 → 智能补货** 中，可 **一键全部转采购申请** 或按行 **转采购申请**（需 `procurement:request` 权限）。

### 6.4 采购订单

路径：**采购管理 → 采购订单**

1. 由申请转换或手工新建。
2. 选择供应商、明细、单价。
3. **审批** 通过后进入可收货状态。
4. 审批时会 **扣减采购预算**（如已配置预算）。

### 6.5 收货验收

路径：**采购管理 → 收货验收**

1. 选择已审批的采购订单。
2. 选择收货仓库、实收数量。
3. 确认后 **库存增加**，并生成 **应付单**。

## 7. 销售管理

### 7.1 业务流程

**订单主线：**

```
销售报价 → 审批 → 转销售订单 → 审批 → 发货（扣库存）→ 应收生成 → 售后（可选）
```

**线索主线（沈阳公海 POI）：**

```
公海浏览/筛选 → 领取至我的线索 → 联系上报（可选录音）→ 主管审核 → 转客户 / 继续跟进
```

### 7.2 客户管理

维护客户档案、联系人、信用额度。

### 7.3 销售报价

1. 新建报价单，添加产品行与报价单价。
2. 审批后可 **转为销售订单**。

### 7.4 销售订单

1. 创建/转换订单，确认客户与明细。
2. 审批通过后可在 **发货** 中选择仓库出库。
3. 发货前系统 **校验库存**；不足则提示失败。
4. 发货成功后生成 **应收单**。

### 7.5 发货管理

对已审批订单执行出库，扣减对应仓库库存。

### 7.6 售后服务

路径：**销售管理 → 售后服务**

- 创建售后工单，关联客户/销售订单。
- 跟踪处理状态（待处理、处理中、已完成等）。

### 7.7 公海线索

路径：**销售管理 → 公海线索**

公海存放尚未分配的销售线索（本系统已导入沈阳市真实 POI 商户数据，含店名、电话、地址、区县、行业分类等）。

![公海线索](images/erp-leads-pool.png)

**页面说明：**

| 元素 | 说明 |
|------|------|
| 已持有 X / 50 条 | 当前用户私海持有量上限为 **50** 条 |
| 区县 / 分类 / 电话 | 按行政区、高德一级行业、是否有电话筛选 |
| 店名/电话/地址 | 关键词模糊搜索 |
| 领取 | 将单条线索划入「我的线索」，进入 **15 天保护期** |
| 批量领取 | 勾选多行后一次性领取（受上限约束） |

**操作步骤：**

1. 使用 `sales_clerk`（销售专员）或具备 `lead:claim` 权限的账号登录。
2. 按需设置筛选项，点击 **查询**。
3. 点击行内 **领取**，或勾选后 **批量领取**。
4. 领取成功后，线索从公海列表消失，出现在 **我的线索**。

> 保护期内的线索仅持有人可跟进；超期未有效跟进将自动回收至公海（系统定时任务）。

### 7.8 我的线索与联系上报

路径：**销售管理 → 我的线索**

![我的线索](images/erp-leads-mine.png)

| 列 | 说明 |
|----|------|
| 保护期 | 剩余天数，到期未跟进将回收到公海 |
| 跟进次数 | 含联系上报与手工跟进记录 |
| 操作 | **联系上报**、跟进、转为客户、释放、标记无效 |

#### 联系上报（结构化跟进）

销售专员完成一次电话/拜访后，应使用 **联系上报** 提交结构化记录（全部进入待审队列，由主管审核）。

![联系上报表单](images/erp-leads-contact-report.png)

**操作步骤：**

1. 在「我的线索」行内点击 **联系上报**，右侧打开表单。
2. 填写必填项：
   - **跟进方式**：电话 / 拜访 / 微信 / 其他
   - **联系结果**（见下表）
   - **沟通摘要**：客户反馈、意向、下次计划等
3. **电话录音（可选）**：点击 **上传录音**，支持 mp3/m4a/wav，单文件 ≤ 10MB，存储于 MinIO。
4. **质量标记**（可选）：有效 / 空号 / 已关店等。
5. 点击 **提交上报**。提交后该条记录状态为 **待审核**，跟进次数 +1。

**联系结果枚举：**

| 结果 | 说明 | 附加要求 |
|------|------|----------|
| 已接通 | 电话接通并完成沟通 | — |
| 未接/挂断 | 无人接听或挂断 | — |
| 有意向 | 客户表达合作意向 | — |
| 明确拒绝 | 客户明确不需要 | — |
| 约下次联系 | 约定下次回访时间 | 必须填写 **下次联系时间** |
| 空号/错号 | 号码无效 | — |
| 已关店 | 商户已停业 | — |

**权限：** `lead:report`（销售专员默认具备）。重新登录后权限才会刷新。

### 7.9 上报审核与联系统计

路径：**销售管理 → 上报审核**（需 `lead:review` 或 `lead:manage`）

销售总监、系统管理员可在此审核全员上报，查看录音与统计数据。销售专员访问该页将显示 **403**。

#### 待审核 / 全部上报

![上报审核](images/erp-leads-reports-pending.png)

| 列 | 说明 |
|----|------|
| 上报人 | 提交上报的销售姓名 |
| 联系结果 | 结构化结果标签 |
| 录音 | 有附件时显示 **播放**（打开 MinIO 预签名下载链接） |
| 审核状态 | 待审核 / 已通过 / 已驳回 |
| 操作 | **通过**、**驳回**（仅待审核行） |

**审核步骤：**

1. 打开 **待审核** 标签页，查看待处理数量。
2. 可选点击 **播放** 收听录音。
3. 点击 **通过** 或 **驳回**，在弹窗中填写 **审核备注**（可选），确认。
4. 审核完成后记录移至 **全部上报** 列表，待审核数量减少。

#### 联系统计

![联系统计](images/erp-leads-reports-stats.png)

| 指标 | 含义 |
|------|------|
| 上报总数 | 全部联系上报条数 |
| 待审核 | 尚未审核条数 |
| 接通率 | 「已接通」占比 |
| 有意向率 | 「有意向」占比 |
| 按结果分布 | 各联系结果数量 |
| 按销售统计 | 各上报人提交量排名 |

### 7.10 公海数据导入（管理员）

路径：**系统管理 → 线索导入**（`/system/leads/import`，需 `lead:import` 权限，`admin` 默认具备）

公海线索有三种导入方式，按数据量选择：

| 场景 | 推荐方式 |
|------|----------|
| 自有 Excel/CRM 导出，几百～几千条 | **页面上传 CSV** 或粘贴 JSON |
| 外部系统定时同步 | **API** `POST /api/leads/import` |
| 沈阳全城 POI、尽量补全电话 | **运维脚本** `import-shenyang-poi.mjs` + 高德 Key |

导入完成后，在 **销售管理 → 公海线索** 查看数量与筛选项是否正常。

#### 方式 A：页面上传 CSV

1. 使用 `admin` 登录 → **系统管理 → 线索导入**。
2. 点击 **上传 CSV**，选择 UTF-8 编码的 CSV 文件。
3. 导入结束后页面显示 **新增 / 跳过（重复）/ 失败** 数量。

**CSV 表头（第一行）：**

```text
name,phone,address,district,category,source,source_id
```

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 店名/商户名 |
| `phone` | 否 | 主联系电话 |
| `address` | 否 | 详细地址 |
| `district` | 否 | 区县，建议与公海筛选项一致（沈阳 13 区县，见下表） |
| `category` | 否 | 行业大类，建议与公海「分类」筛选项一致（见下表） |
| `source` | 否 | 数据来源，默认 `manual`；有外部 ID 时填 `amap`、`tencent` 等 |
| `source_id` | 否 | 外部 POI 编号，与 `source` 配合用于去重 |

**可选扩展列**（表头可用下划线）：`phone_backup`（备用电话）、`lng`、`lat`（经纬度）。

**CSV 示例：**

```csv
name,phone,address,district,category,source,source_id
云之印图文广告,15840454431,黄河北大街25号,皇姑区,生活服务,manual,
恩林房翻新(浑南店),13889173430,浑南区某路88号,浑南区,生活服务,amap,B0FFXXXX
```

**沈阳 13 区县（`district` 建议值）：**  
和平区、沈河区、大东区、皇姑区、铁西区、苏家屯区、浑南区、沈北新区、于洪区、辽中区、新民市、康平县、法库县。

**行业大类（`category` 建议值，与公海筛选一致）：**  
餐饮服务、购物服务、生活服务、体育休闲服务、医疗保健服务、住宿服务、汽车服务、汽车销售、汽车维修、摩托车服务、科教文化服务、金融保险服务、公司企业、商务住宅、风景名胜、公共设施、其他。

#### 方式 B：页面粘贴 JSON

同一路径下方 **JSON 文本框**，格式示例：

```json
[
  {
    "name": "示例餐饮店",
    "phone": "024-12345678",
    "address": "和平区某某路1号",
    "district": "和平区",
    "category": "餐饮",
    "source": "manual"
  }
]
```

点击 **导入 JSON** 提交。字段与 CSV 相同，亦支持 `phoneBackup`、`lng`、`lat`、`remark` 等。

#### 方式 C：API 批量导入

适合脚本或第三方系统对接：

```http
POST /api/leads/import
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "items": [
    {
      "name": "示例店",
      "phone": "13800001111",
      "district": "铁西区",
      "category": "购物服务"
    }
  ]
}
```

响应示例：`{ "created": 100, "skipped": 5, "failed": 0, "errors": [] }`  
`skipped` 表示与库内已有记录重复而未写入。

#### 方式 D：沈阳全量 POI 脚本（大批量）

已在本地导入约 2.7 万条沈阳真实 POI 时，使用的是项目根目录运维脚本（直连数据库，不经过页面）：

```powershell
cd backend
node ../scripts/import-shenyang-poi.mjs

# 清空现有公海后重新导入
node ../scripts/import-shenyang-poi.mjs --replace

# 仅高德 / 仅 OpenStreetMap
node ../scripts/import-shenyang-poi.mjs --amap-only
node ../scripts/import-shenyang-poi.mjs --osm-only
```

**前置条件：**

- PostgreSQL 可连接，已执行 migrate 与 seed。
- `backend/.env` 中配置 `AMAP_API_KEY`（[高德开放平台](https://lbs.amap.com/) 申请 **Web 服务** Key）。
- 可选：导入后执行 `node scripts/recategorize-leads.mjs` 统一行业分类。

#### 去重规则

系统写入前自动去重，重复记录会计入 **跳过**，不会覆盖已有数据：

| 条件 | 去重键 |
|------|--------|
| 填写了 `source` + `source_id`，且 `source` 不是 `manual` | 店名 + 来源 + 外部 ID |
| 手工导入或未填外部 ID | 店名 + 电话 + 区县 |

#### 导入后验证

1. **销售管理 → 公海线索**：列表有数据，区县/分类筛选可用。
2. **销售管理 → 线索统计**（如有权限）：查看公海/已领取数量分布。
3. 用 `sales_clerk` 试领 1 条，确认 **我的线索** 与 **联系上报** 流程正常。

## 8. 库存管理

### 8.1 库存台账

路径：**库存管理 → 库存台账**

查看各仓库、各产品的当前库存数量。支持按产品、仓库筛选。

### 8.2 出入库管理

- **入库/出库调整**：手工修正库存（需 `inventory:inout`）。
- 填写产品、仓库、数量、类型（调整/其他）。

### 8.3 库存调拨

路径：**库存管理 → 库存调拨**

在 two 仓库之间转移库存，源仓库减少、目标仓库增加。

### 8.4 库存盘点

路径：**库存管理 → 库存盘点**

1. 创建盘点单，选择仓库。
2. 录入实盘数量。
3. **完成盘点** 后系统自动生成差异调整。

### 8.5 库存预警

路径：**库存管理 → 库存预警**

列出当前库存 **低于安全库存** 的产品。安全库存在 **产品管理** 中维护。

### 8.6 产品与仓库主数据

- **产品管理**：编码、名称、单位、单价、安全库存、是否序列号管理。
- **仓库管理**：仓库编码、名称、地址。

### 8.7 批次 / 序列号追溯

路径：**库存管理 → 批次追溯**

功能：

- **批次追溯**：输入批次号，查看库存记录与流转。
- **序列号追溯**：输入序列号，查看全生命周期。
- **上链存证 / 链上验真**：将批次数据锚定到哈希链（可选公链 RPC）。
- **注册序列号**：为新序列号建立档案。

区块链 RPC 未配置时使用 **本地哈希链**；配置 `BLOCKCHAIN_RPC_URL` 后写入公链交易哈希。

## 10. 财务管理

### 10.1 总账

- **科目管理**：维护会计科目（资产/负债/权益/收入/费用）。
- **凭证管理**：录入借贷分录，**过账** 后影响科目余额。

### 10.2 应收管理

- 由 **销售发货** 自动生成应收单。
- 登记 **收款**，更新已收金额与状态。

### 10.3 应付管理

- 由 **采购收货** 自动生成应付单。
- 登记 **付款**。

### 10.4 固定资产

- 资产卡片：原值、使用年限、存放地点。
- **计提折旧**：按直线法计算并生成凭证。
- **处置**：报废或出售。

### 10.5 预算管理

- 按年度、部门、类别设置预算总额。
- 采购订单审批时 **自动扣减** 已用预算。

### 10.6 财务报表

路径：**财务管理 → 财务报表**

包含应收/应付汇总、固定资产净值、科目余额表等（需 `finance:report` 或 `report:center`）。

## 17. 常见问题

### Q1：登录提示用户名或密码错误？

- 确认租户为 `default`。
- 确认密码与 `backend/.env` 中 seed 配置一致。
- 检查后端是否运行、数据库是否已 seed。

### Q2：菜单项缺失？

- 当前角色无对应权限；联系管理员在 **角色管理** 中分配。
- 重新登录使权限生效。

### Q3：访问页面显示 403？

- 直接输入了无权限的路由；从侧栏进入有权限的页面即可。

### Q4：发货提示库存不足？

- 在 **库存台账** 检查对应仓库数量。
- 通过采购收货或生产完工补充库存。

### Q5：库存预警无数据？

- 在 **产品管理** 设置 **安全库存 > 0**，且当前库存低于该值。

### Q6：报表/B  I 数据不是最新？

- 报表缓存约 60 秒；等待或强刷页面。
- 生产环境配置 Redis 后多实例共享缓存。

### Q7：权限保存后未生效？

- 用户需 **退出并重新登录**。
- 确认保存时无报错（权限列表接口正常）。

### Q8：区块链上链失败？

- 未配置 RPC 时仅本地哈希链，属正常。
- 配置 `BLOCKCHAIN_RPC_URL` 与 `BLOCKCHAIN_PRIVATE_KEY` 后重试。

### Q9：联系上报按钮灰色或提交失败？

- 确认当前账号持有该线索（**我的线索** 中操作，非公海）。
- 销售专员需具备 `lead:report` 权限；修改权限后 **重新登录**。
- 选择「约下次联系」时必须填写 **下次联系时间**。

### Q10：上报审核页无数据或显示 403？

- **403**：当前角色无 `lead:review`；销售专员只能上报，不能审核。
- **无数据**：待审核为空表示已全部处理；可切换 **全部上报** 查看历史。
- 页面首次加载若 token 未就绪，**刷新** 即可。

### Q11：录音上传或播放失败？

- 确认 MinIO 已启动（Docker：`docker compose` 中 `minio` 服务）。
- 检查 `backend/.env` 中 `MINIO_ENDPOINT`、`MINIO_ACCESS_KEY`、`MINIO_SECRET_KEY`。
- 仅支持音频格式，单文件 ≤ 10MB。

### Q12：公海领取提示已达上限？

- 单人最多同时持有 **50** 条线索；请先 **释放** 或 **转为客户** 后再领取。

### Q13：如何向公海批量导入线索？

- 小批量：管理员登录 → **系统管理 → 线索导入**，上传 CSV 或粘贴 JSON（详见 [7.10 公海数据导入](#710-公海数据导入管理员)）。
- 大批量（沈阳 POI）：配置 `AMAP_API_KEY` 后运行 `scripts/import-shenyang-poi.mjs`。
- 接口对接：调用 `POST /api/leads/import`。
- 重复数据会自动 **跳过**，不会覆盖已有记录。
