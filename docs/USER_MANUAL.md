## 1. 系统概述

本 ERP 系统覆盖企业核心业务流程：

| 模块 | 主要能力 |
|------|----------|
| 采购 | 供应商、采购申请、采购订单、收货验收 |
| 销售 | 客户、报价、销售订单、发货、售后工单 |
| 库存 | 台账、出入库、调拨、盘点、预警、批次/序列号追溯 |
| 生产 | BOM、生产计划、工单、质检 |
| 财务 | 总账、应收/应付、固定资产、预算、财务报表 |
| 人力 | 员工、部门、岗位、考勤、薪资、绩效 |
| 项目 | 项目与任务进度 |
| 报表 | 运营概览、BI 大屏、智能补货/财务分析 |
| 系统 | 用户、角色、权限、租户、审计、文件、集成 |

技术栈：React 前端 + NestJS 后端 + PostgreSQL，支持 Redis 缓存、MinIO 文件存储、WebSocket 通知。

## 2. 访问与登录

### 2.1 访问地址

| 环境 | 前端地址 | 后端 API |
|------|----------|----------|
| 开发模式 | http://localhost:5173 | http://localhost:3000/api（Vite 代理） |
| Docker Compose | http://localhost:8080 | http://localhost:3000/api |

### 2.2 登录步骤

1. 打开浏览器访问上述前端地址，未登录时会自动跳转到登录页。
2. 填写 **租户**（默认选 `default` / 默认租户）。
3. 填写 **用户名** 和 **密码**。
4. 点击 **登录**。

登录页右上角可切换 **中文 / English**。

### 2.3 默认与演示账号

执行数据库 seed 后会自动创建管理员与多角色演示用户。密码在 `backend/.env` 中通过以下变量配置（**切勿使用默认值部署到公网**）：

| 变量 | 说明 |
|------|------|
| `SEED_ADMIN_PASSWORD` | 管理员 `admin` 的初始密码 |
| `SEED_DEMO_PASSWORD` | 各角色演示用户的统一初始密码 |

| 账号 | 角色 | 用途 |
|------|------|------|
| `admin` | 系统管理员 | 全功能管理 |
| `employee` | 普通员工 | 权限测试（采购申请、库存只读、报表） |
| `finance_director` | 财务总监 | 财务全盘 |
| `finance_clerk` | 财务专员 | 日常记账 |
| `procurement_director` | 采购总监 | 采购全流程 |
| `procurement_clerk` | 采购专员 | 采购执行 |
| `sales_director` | 销售总监 | 销售全盘 |
| `sales_clerk` | 销售专员 | 报价/订单/发货 |
| `production_manager` | 生产经理 | 生产+库存协调 |
| `production_clerk` | 生产专员 | 工单执行 |
| `warehouse_admin` | 仓库管理员 | 库存与追溯 |
| `hr_clerk` | 人事专员 | 人事模块 |

> seed 完成后控制台会列出已创建的用户名；密码以 `.env` 配置为准。

## 3. 界面布局与通用操作

登录成功后进入主界面，由四部分组成：

![数据看板](images/erp-dashboard.png)

### 3.1 左侧导航栏（Sider）

- 按业务域分组：财务、采购、销售、生产、库存、人力等。
- **仅显示当前用户有权限的菜单**；无权限的模块不会出现。
- 窄屏（手机/平板）下自动变为 **Drawer 抽屉菜单**，点击左上角图标展开。

### 3.2 顶栏（Header）

| 元素 | 说明 |
|------|------|
| 面包屑 | 显示当前页面路径，如 `首页 / 采购管理 / 采购申请` |
| 区域设置 | `Shanghai · CNY` 按钮，切换时区与币种（会同步到用户 Profile） |
| 语言 | 中文 / English |
| 通知铃铛 | 站内通知列表，支持 WebSocket 实时推送 |
| 用户名 | 当前登录用户姓名，点击可退出登录 |

### 3.3 主内容区

- 页面标题由 **PageTitle** 组件统一展示。
- 列表页普遍采用 **ResponsiveTable**：宽屏表格、窄屏自动卡片化，便于手机浏览。
- 常见按钮：**新建**、**编辑**、**删除**、**审批**、**刷新** 等，操作结果以右上角 Message 提示。

### 3.4 通用表格操作

1. 点击 **新建** 打开表单弹窗（Modal）。
2. 填写必填项（带红色星号），点击 **保存/创建**。
3. 行内 **编辑** 修改记录；**删除** 需二次确认。
4. 带审批流的单据需先 **提交/审批** 才能进入下一环节。

### 3.5 退出登录

点击 Header 右侧用户名 → **退出登录**。Token 清除后需重新登录。

## 4. 角色、账号与权限

### 4.1 权限模型

- 系统预置 **41 项权限**，按模块分组（system、finance、procurement、sales 等）。
- 每个 **角色** 绑定若干权限；用户通过 **角色** 获得权限。
- API 与菜单均受权限控制：
  - **菜单**：侧栏隐藏无权限项。
  - **路由**：直接访问无权限 URL 显示 403 页面。
  - **接口**：无权限返回 HTTP 403。

### 4.2 预置角色（12 个）

| 角色 | 权限数 | 典型使用人 |
|------|--------|------------|
| 系统管理员 | 41（全部） | IT / 实施顾问 |
| 财务总监 | 7 | 财务负责人 |
| 财务专员 | 5 | 会计 |
| 采购总监 | 7 | 采购经理 |
| 采购专员 | 4 | 采购员 |
| 销售总监 | 7 | 销售经理 |
| 销售专员 | 5 | 销售员 |
| 生产经理 | 8 | 车间主任 |
| 生产专员 | 4 | 计划员/操作工 |
| 仓库管理员 | 5 | 仓管 |
| 人事专员 | 5 | HR |
| 普通员工 | 4 | 一线员工 |

### 4.3 配置角色权限

路径：**系统管理 → 角色管理**

![角色管理](images/erp-role-management.png)

**操作步骤：**

1. 在列表中找到目标角色，点击 **权限** 按钮。
2. 在弹窗中按模块勾选权限（finance、inventory、sales 等分组展示）。
3. 点击 **确定** 保存；系统会替换该角色的全部权限绑定。
4. 已登录用户需 **重新登录** 后权限才会刷新（JWT 内不含实时权限变更）。

**新建角色：**

1. 点击 **+ 添加角色**。
2. 填写角色名称、描述 → 创建。
3. 再点击 **权限** 分配具体权限。

### 4.4 用户管理

路径：**系统管理 → 用户管理**

1. **新建用户**：用户名、姓名、邮箱、角色、初始密码。
2. **编辑**：可修改角色、状态（启用/禁用）。
3. 用户被禁用后无法登录。

### 4.5 权限测试建议

用 `employee` 账号（演示密码见 `.env` 中 `SEED_DEMO_PASSWORD`）登录验证：

- 侧栏应仅有：采购申请、部分库存、报表中心等。
- 访问 `/system/role` 应显示 403。
- 可创建采购申请；**不能** 执行出入库调整（403）。

运行自动化审计（可选）：

```powershell
node tests/permissions-audit.mjs
```

## 5. 数据看板与 BI 大屏

路径：**数据看板** 或 `/dashboard`

- **指标卡片**：销售总额、采购订单数、库存总量、系统用户数。

## 6. 采购管理

### 6.1 业务流程总览

```
采购申请 → 审批 → 转采购订单 → 审批采购单 → 收货入库 → 应付生成
```

### 6.2 供应商管理

路径：**采购管理 → 供应商管理**

- 维护供应商编码、名称、联系人、电话。
- 采购订单创建时需选择供应商。

### 6.3 采购申请

路径：**采购管理 → 采购申请**

![采购申请](images/erp-procurement.png)

**新建申请：**

1. 点击 **+ 新建申请**。
2. 填写标题、申请原因。
3. 添加明细行：选择产品、填写数量、预估单价（可选）。
4. 保存后状态为 **草稿**，可编辑。
5. 点击 **提交审批** → 审批通过后可用 **转采购单** 生成采购订单。

**智能补货转申请：**

在 **报表中心 → 智能分析 → 智能补货** 中，可 **一键全部转采购申请** 或按行 **转采购申请**（需 `procurement:request` 权限）。

### 6.4 采购订单

路径：**采购管理 → 采购订单**

1. 由申请转换或手工新建。
2. 选择供应商、明细、单价。
3. **审批** 通过后进入可收货状态。
4. 审批时会 **扣减采购预算**（如已配置预算）。

### 6.5 收货验收

路径：**采购管理 → 收货验收**

1. 选择已审批的采购订单。
2. 选择收货仓库、实收数量。
3. 确认后 **库存增加**，并生成 **应付单**。

## 7. 销售管理

### 7.1 业务流程

```
销售报价 → 审批 → 转销售订单 → 审批 → 发货（扣库存）→ 应收生成 → 售后（可选）
```

### 7.2 客户管理

维护客户档案、联系人、信用额度。

### 7.3 销售报价

1. 新建报价单，添加产品行与报价单价。
2. 审批后可 **转为销售订单**。

### 7.4 销售订单

1. 创建/转换订单，确认客户与明细。
2. 审批通过后可在 **发货** 中选择仓库出库。
3. 发货前系统 **校验库存**；不足则提示失败。
4. 发货成功后生成 **应收单**。

### 7.5 发货管理

对已审批订单执行出库，扣减对应仓库库存。

### 7.6 售后服务

路径：**销售管理 → 售后服务**

- 创建售后工单，关联客户/销售订单。
- 跟踪处理状态（待处理、处理中、已完成等）。

## 8. 库存管理

### 8.1 库存台账

路径：**库存管理 → 库存台账**

查看各仓库、各产品的当前库存数量。支持按产品、仓库筛选。

### 8.2 出入库管理

- **入库/出库调整**：手工修正库存（需 `inventory:inout`）。
- 填写产品、仓库、数量、类型（调整/其他）。

### 8.3 库存调拨

路径：**库存管理 → 库存调拨**

在 two 仓库之间转移库存，源仓库减少、目标仓库增加。

### 8.4 库存盘点

路径：**库存管理 → 库存盘点**

1. 创建盘点单，选择仓库。
2. 录入实盘数量。
3. **完成盘点** 后系统自动生成差异调整。

### 8.5 库存预警

路径：**库存管理 → 库存预警**

列出当前库存 **低于安全库存** 的产品。安全库存在 **产品管理** 中维护。

### 8.6 产品与仓库主数据

- **产品管理**：编码、名称、单位、单价、安全库存、是否序列号管理。
- **仓库管理**：仓库编码、名称、地址。

### 8.7 批次 / 序列号追溯

路径：**库存管理 → 批次追溯**

功能：

- **批次追溯**：输入批次号，查看库存记录与流转。
- **序列号追溯**：输入序列号，查看全生命周期。
- **上链存证 / 链上验真**：将批次数据锚定到哈希链（可选公链 RPC）。
- **注册序列号**：为新序列号建立档案。

区块链 RPC 未配置时使用 **本地哈希链**；配置 `BLOCKCHAIN_RPC_URL` 后写入公链交易哈希。

## 9. 生产管理

### 9.1 BOM 管理

维护成品与组件的物料清单（Bill of Materials），定义单位用量。

### 9.2 生产计划

制定计划单，关联产品与计划数量、日期。

### 9.3 工单管理

```
下达 → 开工 → 领料 → 完工入库
```

- **下达**：由计划或手工创建工单。
- **开工**：记录实际开始时间。
- **完工**：成品入库，可消耗 BOM 组件库存。

### 9.4 质量管理

记录质检单：合格/不合格数量、检验员、关联工单。

## 10. 财务管理

### 10.1 总账

- **科目管理**：维护会计科目（资产/负债/权益/收入/费用）。
- **凭证管理**：录入借贷分录，**过账** 后影响科目余额。

### 10.2 应收管理

- 由 **销售发货** 自动生成应收单。
- 登记 **收款**，更新已收金额与状态。

### 10.3 应付管理

- 由 **采购收货** 自动生成应付单。
- 登记 **付款**。

### 10.4 固定资产

- 资产卡片：原值、使用年限、存放地点。
- **计提折旧**：按直线法计算并生成凭证。
- **处置**：报废或出售。

### 10.5 预算管理

- 按年度、部门、类别设置预算总额。
- 采购订单审批时 **自动扣减** 已用预算。

### 10.6 财务报表

路径：**财务管理 → 财务报表**

包含应收/应付汇总、固定资产净值、科目余额表等（需 `finance:report` 或 `report:center`）。

## 13. 报表与智能分析

### 13.1 报表中心

路径：**报表中心**

汇总运营数据，与 Dashboard 类似，提供更详细的报表入口。

### 13.2 智能分析

路径：**报表中心 → 智能分析**

![智能分析](images/erp-intelligence.png)

#### 智能补货 Tab

根据 **安全库存** 与 **近 30 天出库速度** 计算：

| 字段 | 含义 |
|------|------|
| 建议采购量 | 综合缺口与 lead time 需求量 |
| 日均消耗 | 近 30 天出库 ÷ 30 |
| 可覆盖天数 | 当前库存 ÷ 日均消耗 |
| 优先级 | 高 / 中 / 低 |

操作：

- **一键全部转采购申请**：为所有建议行生成一张采购申请。
- 行内 **转采购申请**：仅为该产品生成申请。

#### 智能财务 Tab

展示 AI 规则生成的财务洞察：应收应付对比、净头寸、毛利率提示、资产负债表快照等。

## 14. 系统管理

### 14.1 系统配置

- 公司信息、系统时区、数据字典（订单状态等）。

### 14.2 文件中心

上传附件至 MinIO（需配置 `MINIO_*` 环境变量），支持下载预签名链接。

### 14.3 系统集成

REST 导出接口，供外部系统拉取主数据/订单/库存（需 `integration:sync`）。

### 14.4 审计日志

查询登录、操作、权限拒绝记录，支持按用户、时间、类别筛选。

### 14.5 租户管理

多租户环境下创建租户；登录时选择对应 **租户编码**。

## 15. 区域设置与国际化

### 15.1 语言切换

- 登录页或 Header 右上角切换 **中文 / English**。
- 菜单、面包屑、通用按钮已双语；部分业务字段仍可能为中文。

### 15.2 时区与币种

1. 点击 Header **Shanghai · CNY** 区域按钮。
2. 选择时区（如 `Asia/Shanghai`、`America/New_York`）。
3. 选择币种（CNY / USD / EUR / JPY / GBP）。
4. 设置会 **保存到用户 Profile**（`PATCH /api/auth/me/preferences`），下次登录自动恢复。

## 16. 移动端与离线（PWA）

- 支持安装为 **渐进式 Web 应用**（浏览器「安装应用」）。
- 离线时顶部显示 **离线横幅**，部分功能不可用。
- 新版本可用时提示 **刷新** 以加载最新前端。
- 表格与布局针对窄屏优化（ResponsiveTable + Drawer 侧栏）。

## 17. 常见问题

### Q1：登录提示用户名或密码错误？

- 确认租户为 `default`。
- 确认密码与 `backend/.env` 中 seed 配置一致。
- 检查后端是否运行、数据库是否已 seed。

### Q2：菜单项缺失？

- 当前角色无对应权限；联系管理员在 **角色管理** 中分配。
- 重新登录使权限生效。

### Q3：访问页面显示 403？

- 直接输入了无权限的路由；从侧栏进入有权限的页面即可。

### Q4：发货提示库存不足？

- 在 **库存台账** 检查对应仓库数量。
- 通过采购收货或生产完工补充库存。

### Q5：库存预警无数据？

- 在 **产品管理** 设置 **安全库存 > 0**，且当前库存低于该值。

### Q6：报表/B  I 数据不是最新？

- 报表缓存约 60 秒；等待或强刷页面。
- 生产环境配置 Redis 后多实例共享缓存。

### Q7：权限保存后未生效？

- 用户需 **退出并重新登录**。
- 确认保存时无报错（权限列表接口正常）。

### Q8：区块链上链失败？

- 未配置 RPC 时仅本地哈希链，属正常。
- 配置 `BLOCKCHAIN_RPC_URL` 与 `BLOCKCHAIN_PRIVATE_KEY` 后重试。
