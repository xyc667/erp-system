# ERP 外勤 App 使用说明

> 适用版本：**1.1.0**（Android）· iOS 内测版见 TestFlight  
> 配套 Web 说明：[详细使用说明](USER_MANUAL.md) · [简明用户指南](USER_GUIDE.md)  
> iOS 打包指南：[FIELD_APP_IOS_SETUP.md](FIELD_APP_IOS_SETUP.md)

---

## 1. 应用概述

**ERP 外勤** 是面向销售外勤的移动端（Android / iOS），与 Web ERP 共用同一套账号与后端。主要解决「出门跑店」场景下的线索领取、联系上报、导航到店与消息提醒。

| 能力 | App | Web ERP |
|------|-----|---------|
| 公海浏览 / 领取 / 批量领取 | ✅ | ✅ |
| 我的线索 / 拨号 / 导航 | ✅ | ✅ |
| 联系上报 + 上传录音 | ✅ | ✅ |
| 我的上报记录 | ✅ | — |
| 上报审核（通过/驳回） | ✅（主管） | ✅ |
| 联系统计大屏 | — | ✅ |
| 报价 / 订单 / 发货 | — | ✅ |
| 消息通知 | ✅ | ✅ |

## 2. 安装与登录

### 2.1 Android：安装 APK

1. 手机与部署 ERP 的电脑处于 **同一局域网**（WiFi）。
2. 浏览器打开（示例，以实际 IP 为准）：
   ```
   http://192.168.3.4:3001/apk/erp-field-latest.apk
   ```
3. 下载完成后安装；首次安装需允许 **「未知来源应用」**。
4. 若提示签名冲突，先 **卸载旧版** 再安装。

> 管理员可在 `backend/public/apk/` 替换 `erp-field-latest.apk`，并更新 `backend/.env` 中的版本号后重启后端。

### 2.2 iOS：TestFlight 内测

iOS 不能像 Android 一样直接下载 APK，需通过 **TestFlight** 安装：

1. 在 App Store 安装 **TestFlight**。
2. 打开管理员提供的 TestFlight 邀请链接（示例）：
   ```
   https://testflight.apple.com/join/XXXXXXXX
   ```
3. 在 TestFlight 中安装 **ERP 外勤**。
4. 有新版本时，App 内会提示 **「前往 TestFlight 更新」**。

> 管理员配置见 [FIELD_APP_IOS_SETUP.md](FIELD_APP_IOS_SETUP.md)。

### 2.3 登录

| 字段 | 说明 |
|------|------|
| 租户编码 | 默认 `default` |
| 用户名 | 如 `sales_clerk`（销售）、`sales_director`（主管） |
| 密码 | 与 Web 相同，见 `backend/.env` → `SEED_DEMO_PASSWORD` |

登录页底部会显示当前 API 地址与客户端版本号，便于内网联调排错。

### 2.4 启动与版本检查

- 首次启动会 **检查版本**；有新版本时弹窗提示下载。
- 若后台开启 **强制更新**，必须安装新版本才能进入。
- 内网调试时若连不上服务器，可选择 **「跳过检查」**。

## 3. 界面导航

登录后底部为 **Tab 栏**（按角色显示不同 Tab）：

| Tab | 角色 | 功能 |
|-----|------|------|
| **公海** | 销售 | 浏览、筛选、领取、批量领取、导航 |
| **我的线索** | 销售 | 已领取线索、拨号、导航、联系上报 |
| **我的上报** | 销售（`lead:report`） | 本人提交记录、审核状态、播放录音 |
| **上报审核** | 主管（`lead:review`） | 待审核列表、通过/驳回、播放录音 |
| **消息** | 全部 | 系统通知、未读数、全部已读 |

右上角显示 **姓名** 与 **退出**。

---

## 4. 公海线索

### 4.1 浏览与筛选

1. 进入 **公海** Tab。
2. 顶部显示 **已持有 X / 50 条** 配额。
3. 点 **筛选** 可设置：
   - 关键词（店名 / 电话 / 地址）
   - 区县、行业分类
   - 是否有电话
4. 点 **查询** 刷新列表；下拉可 **刷新**。

### 4.2 单条领取

在线索卡片上点 **领取** → 成功后线索进入 **我的线索**。

### 4.3 批量领取

1. 点右上角 **批量** 进入选择模式。
2. 点击卡片勾选（最多 **10 条**）。
3. 点 **批量领取**。
4. 点 **取消批量** 退出选择模式。

> 单人同时最多持有 **50 条**；达上限时需先在 Web 或 App 释放/转化后再领。

### 4.4 导航（公海）

有 **地址或经纬度** 的线索会显示 **导航** 按钮，点击调起手机地图（高德等）导航到店。

---

## 5. 我的线索

1. 进入 **我的线索** Tab，查看已领取列表。
2. 卡片信息：店名、电话、区县、分类、地址、**保护期剩余天数**（≤3 天标红）。
3. 操作：
   - **点击电话号码** → 系统拨号
   - **导航** → 地图导航到店
   - **联系上报** → 进入上报页

---

## 6. 联系上报

从 **我的线索** 进入 **联系上报** 页。

### 6.1 必填项

| 字段 | 说明 |
|------|------|
| 跟进方式 | 电话 / 拜访 / 微信 / 其他 |
| 联系结果 | 已接通、未接听、有意向、无意向、约下次联系、空号/错号、已关店 |
| 沟通摘要 | 客户反馈、意向、下次计划等 |

选 **约下次联系** 时须填写 **下次联系时间**。

### 6.2 电话录音（可选）

- 点 **选择录音文件**，从手机存储选择 mp3 / m4a / wav 等。
- 单文件最大 **10MB**。
- 选后可 **重新选择** 或 **清除**。

> App **不支持应用内录音**，请使用手机自带通话录音或第三方录音 App，再在此上传文件。

### 6.3 提交

点 **提交上报** → 记录进入 **待审核**，主管在 App 或 Web 审核。

### 6.4 离线暂存

无网络时：

- 顶部显示 **红色离线横幅**。
- 仍可填写并 **提交**；系统 **暂存到本地**。
- 恢复网络后 **自动上传**（含录音文件），并提示同步条数。

---

## 7. 我的上报

**我的上报** Tab 查看本人全部上报：

- 按状态筛选：全部 / 待审核 / 已通过 / 已驳回
- 查看摘要、联系结果、提交时间
- **驳回** 时显示驳回原因
- 有录音时点 **播放录音**（系统播放器打开）
- 可 **地图导航到店**（若线索有地址/坐标）

---

## 8. 上报审核（主管）

具备 `lead:review` 或 `lead:manage` 权限的账号（如 `sales_director`）可见 **上报审核** Tab。

1. 查看 **待审核** 列表（上报人、店名、结果、摘要）。
2. 有录音时点 **播放录音** 收听。
3. 点 **通过** 或 **驳回**，可填审核意见。
4. 审核后销售会在 **消息** Tab 收到通知。

> Web 端 **销售管理 → 上报审核** 功能相同，并含 **联系统计** 大屏；复杂统计建议在 Web 查看。

---

## 9. 消息通知

**消息** Tab 展示系统通知：

| 事件 | 接收人 |
|------|--------|
| 销售提交联系上报 | 主管 |
| 上报通过 / 驳回 | 提交人 |

- 顶部显示 **未读数**，可 **全部已读**。
- 下拉刷新列表。

Web 端 Header **铃铛** 也会收到相同通知；点击上报类通知可跳转审核页。

---

## 10. 地图导航

**导航** 按钮出现在：公海、我的线索、我的上报（关联门店）。

| 数据 | 行为 |
|------|------|
| 有经纬度 | 优先精确导航 |
| 仅有地址 | 按「区县 + 地址 + 店名」搜索导航 |
| 都没有 | 提示无法导航 |

请确保手机已安装 **高德地图** 或其他地图 App。

---

## 11. 版本更新

- 启动时自动检查 `GET /api/app/field-android/latest`。
- **可选更新**：弹窗「立即更新 / 稍后」。
- **强制更新**：必须下载安装新版本。

管理员发布新版本：

1. 构建新 APK 至 `backend/public/apk/erp-field-latest.apk`
2. 更新 `backend/.env` 中 `FIELD_ANDROID_VERSION_CODE` 等
3. 重启后端

## 13. 常见问题

### Q1：登录失败 / 无法连接服务器？

- 确认手机与服务器 **同一 WiFi**。
- 确认后端已启动，端口一般为 **3001**。
- APK 构建时的 API 地址（`EXPO_PUBLIC_API_URL`）须为电脑 **局域网 IP**，不能用 `localhost`。
- 登录页底部可核对当前 API 地址。

### Q2：安装提示签名不一致？

- 卸载旧版后重新安装，或使用与旧版 **相同 keystore** 签名的 APK。

### Q3：领取提示已达上限？

- 单人最多 **50 条**；请释放或转客户后再领。

### Q4：批量领取为什么只能选 10 条？

- 与 Web 一致，单次批量上限 **10 条**；可多次操作。

### Q5：看不到「上报审核」Tab？

- 仅主管角色（`lead:review`）可见；销售专员请用 Web 或 App **我的上报** 查看自己的记录。

### Q6：录音播放失败？

- 确认 MinIO 已配置且后端存储正常（见 [USER_MANUAL Q11](USER_MANUAL.md#q11录音上传或播放失败)）。
- 录音为预签名链接，需在 **网络正常** 时播放。

### Q7：离线提交后多久同步？

- 恢复网络后 **自动同步**；也可重启 App 触发。同步成功会弹窗提示。

### Q8：App 与 Web 数据是否一致？

- **是**，共用同一后端；任一端的领取、上报、审核即时生效。
